WHOI Quantification Results

Quantification Methods

In the chart we can compare the results for each of the methods tested. Each method tries to predict the prevalence for each class in all the files used for testing. Files are in the X-axis while prevalence is in the Y-axis. We have compared the following methods:

  • CC: Classify&Count
  • AC: Adjusted Count
  • PCC: Probabilistic Classify&Count
  • PAC: Probabilistic Adjusted Count
  • HDy: Method based on the Hellinger Distance
  • EM: Method based on the EM algorithm

Features used

Each of these methods has been tested using three different features sets:

  • NF: Normal features downloaded from IFCB dashboard
  • DF: Attributes computed using resnet-18 trained for the ImageNet problem. No fine-tunning has been made to adapt the network to the plankton problem.
  • DFFT: Attributes computed using resnet-18 rained for the ImageNet problem. The difference with the DF aproach is that the network has been fine-tunned with plankton images.

One example: So, if we want to compare the performance of the CC method, using the IFCB features with the CC method using the resnet-18 features without fine-tuning we would have to set visible the following traces: CC_NF and CC_DF. Note that the True prevalence is visible by default.
